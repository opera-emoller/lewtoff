# Phase 4 — Floor 1 Implementation Plan

> No worktrees. Work on `main`.

**Goal:** Literal port of libvorbis 1.3.7's floor 1 fitter + encoder from `~/Documents/src/libvorbis/lib/floor1.c` (1086 LOC) to `src/floor1.rs`. The trickiest port — the heuristic line-fitter has subtle control-flow that any "cleanup" will silently change. Discipline: line-by-line, no "improvements".

**Verification strategy:** No per-kernel C harness this phase (would require constructing a `vorbis_block` + `vorbis_dsp_state`, which is several days of C plumbing for marginal gain). Instead:
1. **Compile + clippy clean** — zero warnings on `cargo clippy --all-targets -- -D warnings`.
2. **Unit tests on math primitives** — pure functions like `render_line`, `render_point`, `searchstep` etc. that have well-defined inputs/outputs and can be tested against hand-computed values.
3. **Setup unpack round-trip** — extend `src/codebook.rs`'s setup-blob unpack to also pull out the floor1 configurations from the same blob, run them through our floor1 setup-decode, assert they don't error.
4. **Phase 9 parity is the ultimate gate** — when end-to-end fails, we'll add targeted floor1 vectors then.

**Architecture:**
- `src/floor1.rs` — port of floor1.c
- The floor1 module exposes: `Floor1Setup` (decoded from the setup blob), `Floor1State` (per-encode state), `Floor1Encoder::forward(spectrum: &[f32]) -> Floor1Output` where `Floor1Output` carries the encoded bits and the floor curve to subtract from the spectrum (residue input).
- The setup blob already contains all floor configurations; Phase 3's `unpack_q5_codebooks` infrastructure extends naturally to also unpack floor1 setup. **Add a sibling `unpack_q5_floors` function in codebook.rs (or split into a new `setup.rs` if it's getting crowded — judgment call).**

**Source of truth:**
- `~/Documents/src/libvorbis/lib/floor1.c`
  - `floor1_info_unpack` — read floor1 setup from setup-header bits (already in blob)
  - `floor1_look` — build the per-block lookup struct
  - `floor1_forward` — the encoder entry point
  - `accumulate_fit`, `fit_line`, `inspect_error` — heuristic fitter
  - `render_line`, `render_point` — bit-exact line interpolation (used by both encode and decode)
- `~/Documents/src/libvorbis/lib/floor1.h` — for struct fields
- Vorbis I §7 (floor 1) — the spec

### Task 4.3: Port `floor1_forward` (the encoder)

THIS is the line-by-line port that needs the literal discipline. ~600 LOC of dense heuristic code.

- [ ] **Step 1: Read `floor1_forward` end-to-end.** Understand the pipeline: log spectrum → bark weighting → greedy line fit → quantize posts → emit bits via codebooks. Don't simplify.
- [ ] **Step 2: Port helpers `accumulate_fit`, `fit_line`, `inspect_error`** as private functions in floor1.rs. Match variable names. Suppress clippy (don't restructure).
- [ ] **Step 3: Port `floor1_forward` itself.** Inputs: log magnitude spectrum (`&[f32; N/2]`), the codebook table from Phase 3, the floor1 state. Outputs: the emitted bits (writer-state pattern: pass a `&mut BitWriter`) and the rendered floor curve (used by the residue stage to subtract).
- [ ] **Step 4: Unit-test what's testable** — `accumulate_fit` on a synthetic spectrum, `fit_line` on a simple linear spectrum. The encoder itself can't be byte-tested without the full pipeline; rely on Phase 9.
- [ ] **Step 5: `cargo build` and `cargo clippy --all-targets -- -D warnings` clean. Suppress lints with localized `#[allow(...)]` rather than restructuring code.**
- [ ] **Step 6: Commit** as `Phase 4: floor1 encoder (literal port)`.
